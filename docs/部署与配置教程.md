# 本地部署教程

本文档帮助你在本地环境中启动竞品分析 Agent 协作系统。

## 环境要求

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端运行环境 |
| npm | 9+ | 前端包管理 |

## 第一步：启动后端

### 1.1 创建 Python 虚拟环境

```bash
cd backend
python3 -m venv venv
source venv/bin/activate    # Linux/Mac
# Windows: venv\Scripts\activate
```

### 1.2 安装依赖

```bash
pip install -r requirements.txt
```

> weasyprint 需要系统级依赖（用于 PDF 导出）。如果安装失败：
> - Ubuntu/Debian: `sudo apt install libpango1.0-dev libgdk-pixbuf2.0-dev libffi-dev`
> - Mac: `brew install pango gdk-pixbuf libffi`
> - 如果不需要 PDF 导出功能，可以忽略 weasyprint 的安装错误，系统仍可正常运行

### 1.3 启动后端服务

`.env` 文件已配置好，直接启动即可：

```bash
uvicorn main:app --reload --port 8000
```

看到如下输出即为成功：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
```

验证：浏览器打开 http://localhost:8000/docs 可以看到 API 文档页面。

## 第二步：启动前端

打开一个**新的终端窗口**：

### 2.1 安装依赖

```bash
cd frontend
npm install
```

### 2.2 启动前端服务

```bash
npm run dev
```

看到如下输出即为成功：

```
▲ Next.js 15.x.x
- Local: http://localhost:3000
```

## 第三步：访问系统

浏览器打开 **http://localhost:3000** 即可使用。

## 功能说明

### 创建分析任务

1. 点击首页的「开始分析」按钮
2. 填写表单：
   - **分析目标**（必填）：你想分析的产品或赛道
   - **自身情况描述**（必填）：你的核心能力、资源、市场定位
   - **竞品名称**（可选）：指定竞品，不填则系统自动识别
   - **行业领域**（可选）：所属行业
   - **关注维度**（可选）：重点分析方向
   - **报告篇幅**：精炼版（2000-3000字）或标准版（5000+字）
3. 提交后系统自动执行，可实时查看 DAG 进度和决策日志
4. 执行完成后右侧弹出完整报告，支持复制和导出 PDF

### Agent 工坊

点击 Header 的「Agent 工坊」可配置各 Agent 的模型和 System Prompt：

- 支持为分析师/撰写者配置双模型（交叉验证）
- 修改 Prompt 后保存即生效
- 可在线测试 Prompt 效果

### 历史记录

点击 Header 的「历史」按钮可查看所有分析任务的历史，包含：
- 用户输入的完整信息
- 执行耗时和 Token 消耗
- 报告全文查看和 PDF 导出

## 常见问题

### Q: PDF 导出失败？

确认 weasyprint 的系统依赖已安装。Ubuntu 执行：
```bash
sudo apt install libpango1.0-dev libgdk-pixbuf2.0-dev libffi-dev
```

### Q: 离开页面后任务丢失？

回到首页会自动检测正在运行的任务，点击蓝色提示条即可恢复进度。已完成的任务可在「历史」中查看。


## .env 配置说明

项目已包含配置好的 `.env` 文件，以下是各字段说明（一般无需修改）：

| 字段 | 说明 |
|------|------|
| `MINIMAX_API_KEY` | OpenAI API Key |
| `MINIMAX_BASE_URL` | OpenAI API 地址 |
| `MODEL_LARGE` | 重度任务模型（分析/撰写） |
| `MODEL_SMALL` | 轻度任务模型（编排/采集/审核） |
| `ANTHROPIC_API_KEY` | Claude API Key（交叉验证用） |
| `ANTHROPIC_MODEL` | Claude 模型版本 |
| `CROSS_VALIDATION_ENABLED` | 是否开启双模型交叉验证 |
| `TAVILY_API_KEY` | 网络搜索 API Key |

## 项目结构

```
├── backend/                 # Python 后端
│   ├── agents/             # Agent 引擎（编排、采集、分析、撰写等）
│   ├── api/                # FastAPI 路由
│   ├── harness/            # Agent 运行时（Provider、Runtime、工具）
│   ├── data/               # SQLite 数据持久化
│   ├── export/             # PDF 导出
│   ├── main.py             # 应用入口
│   └── config.py           # 配置管理
├── frontend/               # Next.js 前端
│   ├── app/                # 页面
│   ├── components/         # UI 组件
│   └── lib/                # 工具函数和 Context
└── docs/                   # 文档
```
