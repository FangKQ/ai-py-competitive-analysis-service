6 个专职 Agent

模拟真实数字调研小组，每个 Agent 各司其职

🎯

Orchestrator Agent

任务规划与分解。解析用户需求，识别竞品，生成 DAG 执行计划，协调各 Agent 工作。参考 Claude Code 的 Lead Agent 模式。

🔍

Collector Agent

信息采集专家。通过 Web 搜索和网页抓取，系统化收集竞品公开信息。支持多实例并行采集不同竞品。

📊

Analyst Agent

深度分析引擎。对采集数据进行结构化分析：功能对比矩阵、市场定位、SWOT 分析，输出符合知识 Schema 的结构化结果。

✍️

Writer Agent

报告撰写专家。将结构化分析转化为专业的竞品分析报告，包含执行摘要、详细对比、战略建议。每条结论标注引用。

🔎

Reviewer Agent

质量审查官。从准确性、完整性、引用质量三个维度评分。未达标则触发交叉审查反馈闭环，要求 Writer/Analyst 修订。

📎

Citation Agent

溯源验证专家。验证报告中每条引用的可访问性和相关性，确保结论有据可查。输出引用可靠性评分和验证报告。

核心特性

不仅是分析工具，更是可观测的 Agent 协作平台

📐 DAG 式任务编排

有向无环图驱动的任务流转，支持并行采集、串行分析、条件分支。参考 Google ADK GraphAgent 设计。

🔄 交叉审查闭环

Reviewer Agent 多维评分，未达标自动触发修订循环。最多 2 轮审查，确保报告质量。

🔗 全程溯源

每条分析结论标注来源 URL 和原文片段。Citation Agent 独立验证每条引用的有效性和可靠性。

👁️ 完全可观测

Agent 决策日志实时追踪，SSE 流式推送执行状态。每一步推理过程和工具调用完全透明。

🔧 Harness 五层架构

Runtime 状态机 + Context 记忆管理 + Capability 工具注册 + Governance 权限审计 + Surface DAG 编排。

⚙️ 棘轮机制

Agent 运行时错误自动转化为永久约束规则，防止重复犯错。参考 Claude Code 的 Ratchet 设计。
